# CLAUDE.md

이 파일은 Claude Code가 이 프로젝트에서 작업할 때 참고하는 가이드입니다.

## 프로젝트 개요

서울시 실시간 통합데이터(서울 열린데이터광장 등 공공 API + 자체 크롤링)를 수집·적재·가공하여,
사용자의 자연어 질문에 서울시 데이터 기반으로 응답하는 시스템.

**데이터 도메인 우선순위**
1. 인구혼잡도 (1순위, 우선 구축 대상)
2. 교통
3. 행사
4. 기상

서울 열린데이터광장의 "서울시 실시간 도시데이터(citydata)" API가 인구혼잡도 등 여러 도메인을
지역(장소) 단위로 함께 제공하는 것으로 보이며, 정확한 API 구조(엔드포인트, 응답 스키마, 지역 코드 목록 등)는
아직 파악되지 않았다. 1순위 파이프라인(인구혼잡도) 구현 전에 해당 API 구조를 먼저 조사해야 한다.

## 아키텍처

```
[Scheduler] --(5~10분 주기)--> [Collector/Worker] --raw--> [Cloudflare R2]
                                                                  |
                                                           [Processor]
                                                                  |
                                                              [DuckDB]
                                                                  |
                                                          [API 서버 (FastAPI)]
                                                                  |
                                                          사용자 질문 -> 응답
```

- **Collector(Worker)**: 공공 API(citydata 등)와 자체 크롤러를 주기적으로(5~10분 간격) 폴링하여
  원본(raw) 응답을 그대로 Cloudflare R2에 적재. 수집 주기는 cron 또는 별도 스케줄러로 관리.
- **Processor**: R2에 쌓인 원본 데이터를 정제/정규화하여 DuckDB에 분석 가능한 테이블로 적재.
- **API 서버**: Python FastAPI 기반. DuckDB를 조회하여 사용자 질문에 응답.
  자연어 질의 응답을 위한 LLM 연동 방식은 **미정**(추후 결정 사항).

## 기술 스택

- 언어/프레임워크: Python (FastAPI) + 별도 워커 프로세스(수집/가공)
- 원본 데이터 저장: Cloudflare R2
- 가공 데이터 저장/분석: DuckDB
- 질의응답 LLM: 미정

## 디렉토리 구조 (제안)

```
collectors/    # 공공 API 폴링 및 크롤링 워커
processors/    # R2 원본 데이터 -> DuckDB 가공 파이프라인
api/           # FastAPI 서버 (질의응답 엔드포인트)
data/          # 로컬 개발용 샘플/캐시 데이터 (R2/DuckDB 실데이터는 제외)
```

실제 구현 시작 전 구조가 바뀔 수 있으며, 코드가 추가되면 이 섹션을 실제 구조에 맞게 갱신할 것.

## 미정 사항 (추후 결정 필요)

- citydata API의 정확한 엔드포인트/응답 스키마/지역 코드 목록 — 인구혼잡도 파이프라인 구현 전 조사 필요
- 질의응답에 사용할 LLM/모델
- 수집 스케줄러 구현 방식 (cron, APScheduler, 클라우드 스케줄러 등 미확정)
- 배포 환경

## 개발 컨벤션

- 아직 코드베이스가 없는 초기 단계. 코드가 추가되면 커밋 규칙, 테스트 실행 명령, 환경변수 목록 등을
  이 섹션에 보완할 것.
